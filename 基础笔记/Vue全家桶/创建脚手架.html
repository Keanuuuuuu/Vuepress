# 3.1 初始化脚手架

# 3.1.1 说明

  • Vue脚手架是Vue官方提供的标准化开发工具（开发平台）（用来翻译vue）（Vue是核心技术）

  • 脚手架最新版本是4.x

  • 问题

当name:'School'时会报错，改为name:'SchooL' 可以运行

// ES6的语法直接引入vue
import Vue from 'vue'
// 引入app组件，是所有组件的父组件
import App from './App.vue'
// 关闭vue的生产提示
Vue.config.productionTip = false
// 创建vue实例对象
new Vue({
	el;'#app',
	template:`<App></App>`,
	components:{
		App
	},
})

与

new Vue({
  render: h => h(App),
}).$mount('#app')
  • 由于使用ES6新语法import Vue from 'vue' 引入的是残缺版的vue，导致模板解析器不能用，template报错
  • 第一种解决办法是引用完整的vue.js 其他的都是一堆"乱七八糟"的vue
  • ![屏幕截图 2022-07-29 174045](屏幕截图 2022-07-29 174045.png)
import Vue from './****'
  • 第二种解决办法是render
// render是一个函数，vue帮你调，含有一个参数，参数的类型是function，作用是创建具体的元素，编写具体的元素，将创建的元素返回即可
render(createElement){
    return createElement('h1','你好啊')
}

// render没有用到this，即可写成箭头函数
render:(createElement)=>{ // 左边只有一个参数，可以把括号省略
    return createElment('h1','你好啊')
}

render:createElment=>createElment('h1','你好啊')

# 对那些“乱七八糟”vue文件的解释

  • vue包含两个部分
  • 1、vue核心：生命周期，处理事件等等
  • 2、模板解析器
如果一直使用这种完整版的vue，在解析打包是使用webpack，生成一个非常大的文件夹，包含图片等，一定也得包含vue这个库，这个时候模板解析在这就不合适了，因为这个时候webpack就已经把.vue文件翻译成.js文件了，而且模板都已经解析完了

  • 关于不同版本的Vue：

    ​ 1.vue.js与vue.runtime.xxx.js的区别：

    ​ (1).vue.js是完整版的Vue，包含：核心功能+模板解析器。

    ​ (2).vue.runtime.xxx.js是运行版的Vue，只包含：核心功能；没有模板解析器。

    ​ 2.因为vue.runtime.xxx.js没有模板解析器，所以不能使用template配置项，需要使用

    ​ render函数接收到的createElement函数去指定具体内容。

（两种模块化语言 ES6 common）

# 3.2 ref与props

# 3.2.1 ref

  • 原生操作获取dom节点 使用id之类的选择器，在vue里可以使用ref

    <template>
	<div>
		<h1 v-text="msg" ref="title"></h1>
		<button ref="btn" @click="showDOM">点我输出上方的DOM元素</button>
		<School ref="sch"/>//给组件加ref 组件不是真实的dom节点
	</div>
</template>

<script>
	//引入School组件
	import School from './components/School'

	export default {
		name:'App',
		components:{School},
		data() {
			return {
				msg:'欢迎学习Vue！'
			}
		},
		methods: {
			showDOM(){
				console.log(this.$refs.title) //真实DOM元素
				console.log(this.$refs.btn) //真实DOM元素
				console.log(this.$refs.sch) //School组件的实例对象（vc）
                // $refs就相当于document.getElementById()
			}
		},
	}
</script>

  • 这里的this是vc（VueComponent）其包含$refs属性

  • 可以说ref是id的替代者

# 3.2.2 全新的配置项（el、data...）：props

  • 通过往标签里直接传属性进而展示在页面上

从父组件里往子组件里传入特殊的属性

<Student name="李四" sex="女" :age="18"/> // 这是在app组件里写student标签

这个时候student的data里没有name等属性及其值

data() {
			console.log(this)
			return {
				msg:'我是学生',
				myAge:this.age
			}
		},

单纯这样写会报错，不能在页面上显示name等信息，原理类似与支付与收款，光单纯支付不行，另一方要确认收款，这个时候就用到props

  1. 功能：让组件接收外部传过来的数据

  2. 传递数据：<Student name="xxx"/>

  3. 接收数据：

    1. 第一种方式（只接收）：props:['name']属于确认收款了

    2. 第二种方式（限制类型）：props:{name:String}、:age="18" //：表示v—bind，双引号里面的内容会被当做js表达式，是数字，那他的类型就是number

    3. 第三种方式（限制类型、限制必要性、指定默认值）：

      props:{
 name:{
 type:String, //类型
 required:true, //必要性
 default:'老王' //默认值
 }
}

    • 由于数据是props配置的，输出vc实例对象时，data里只会有原始的msg，不会出现后来配置的name等属性及其值

    • 但是不必担心，在VUE开发者中打开会看到props，与data平级

    • 注意：如果接收的时候props里面不是字符串的话，就是变量或者表达式，但是该组件没有这个名字的表达式或者变量，这个时候就会报错

    备注：props是只读的，比如说写个methods，按钮改变props里面的年龄（这个时候年龄在vc上），这个时候就会报错，Vue底层会监测你对props的修改，如果进行了修改，就会发出警告，若业务需求确实需要修改，那么请复制props的内容到data中一份，然后去修改data中的数据。

data() {
			console.log(this)
			return {
				msg:'我是一个尚硅谷的学生',
				myAge:this.age // 复制
			}
		},
		methods: {
			updateAge(){
				this.myAge++
			}
		},

# 3.2.3 mixin混入

  • 两个组件共享一个配置,可以把多个组件共用的配置提取成一个混入对象
  • 模块化引入混合
<script>
	import {hunhe,hunhe2} from '../mixin'

	export default {
		name:'Student',
		data() {
			return {
				name:'张三',
				sex:'男'
			}
		},
		mixins:[hunhe,hunhe2]
	}
</script>
  • 在外部js里这样写
export const hunhe = {
	methods: {
		showName(){
			alert(this.name)
		}
	},
	mounted() {
		console.log('你好啊！')
	},
}
export const hunhe2 = {
	data() {
		return {
			x:100,
			y:200
		}
	},
}


   全局混入：
   先在main.js里全局引入
   ```import {hunhe,hunhe2} from './mixin'```
   再在mian.js里使用
   ```Vue.mixin(xxx)```

   局部混入：```mixins:['xxx']  ```

# 3.2.5 scoped样式

  • 在各个组件的style部分中写的样式都将会汇总，那么汇总后就有可能产生因样式名相同的类名冲突

  • 作用：让样式在局部生效，防止冲突。写法：<style scoped>

  • 但是在app组件里不适合使用scoped

  1. 组件化编码流程：

    (1).拆分静态组件：组件要按照功能点拆分，命名不要与html元素冲突。

    (2).实现动态组件：考虑好数据的存放位置，数据是一个组件在用，还是一些组件在用：

​ 1).一个组件在用：放在组件自身即可。

​ 2). 一些组件在用：放在他们共同的父组件上（状态提升）。

(3).实现交互：从绑定事件开始。
