# 第二十五节 路由的使用

Vue3之路由篇 🖖（Vue-Router） - 掘金 (juejin.cn) (opens new window)

# 1、安装配置路由

vue-router@3版本能在vue2中使用，vue-rouer@4只能在vue3中使用

安装时要用 npm i vue-router@3

router是一个插件库，安装后要在main.js里use一下

//引入Vue
import Vue from 'vue'
//引入App
import App from './App.vue'
//引入VueRouter
import VueRouter from 'vue-router'
//引入路由器
import router from './router'

//关闭Vue的生产提示
Vue.config.productionTip = false
//应用插件
Vue.use(VueRouter)

//创建vm
new Vue({
	el:'#app',
	render: h => h(App),
	router:router
})

use后全局多了一个新的配置项：router，通常在src文件夹下配置router文件夹，内含一个js文件用于管理路由们

// 该文件专门用于创建整个应用的路由器
import VueRouter from 'vue-router'
//引入组件
import About from '../components/About'
import Home from '../components/Home'

//创建并暴露一个路由器
export default new VueRouter({
	routes:[
		{
			path:'/about',
			component:About
		},
		{
			path:'/home',
			component:Home
		}
	]
})

这个时候在 main.js里 import router from './router' 、router:router

路由才会真正开始生效：

浏览器的地址栏之后也会带上 /#/标志

# 2、实现页面内路由

  • 在App.vue里，原先可能是使用a标签、button标签实现点击页面跳转
    • 底层默认也是把router-link标签通过引入的路由库转化为a标签，这样即使用的是原来a标签的样式，也不会乱，之后学习另一种方式，可控制转换成对应的标签
  • Vue中借助router-link标签实现路由的切换
    • active-class是vue-router模块的router-link组件中的属性，用来做选中样式的切换 ，激活（点击时配置后面的active 这个class类名）
    • a标签是href属性，router-link是to属性，且不能带“.”和文件后缀名
    • 通过router-view指定组件的呈现位置
 <div class="list-group">
     <!-- 原始html中我们使用a标签实现页面的跳转 -->
     <!-- <a class="list-group-item active" href="./about.html">About</a> -->

     <!-- Vue中借助router-link标签实现路由的切换 -->

     <router-link class="list-group-item" active-class="active" to="/about">About</router-link>

</div>

# 3、注意事项

  • 路由组件：靠路由器匹配渲染出的组件
  • 一般组件：需要先引用，再注册，再亲自在template里书写对应组件名才能使用
  • 开发时会把一般组件放进components文件夹里，把路由放进pages文件夹里
  • 通过切换，“隐藏”了的路由组件，默认是被销毁掉的，需要的时候再去挂载。
  • 每个路由的this，即VC实例对象身上多了$route,$router两个属性，前者属于其彼此，各不相同，每个组件都有自己的$route属性，里面存储着自己的路由信息。，后者再整个项目中只有一个 ，整个应用只有一个router，可以通过组件的$router属性获取到。

# 7、路由的params参数

回顾：Ajax的params参数-> 将问号换成“/”并直接带上参数值，不用键值对

用params参数和query差不多不过要是只在路径后面跟上“/”会产生歧义，会以为“/”后面又是路径

/home/message/detail/${m.id}/${m.title}

需要在路径配置里面令将对应的path补充完成

path:'detail/:id/:title',

  • detail后面是占位符
<!-- 跳转路由并携带params参数，to的字符串写法 -->
<!-- <router-link :to="`/home/message/detail/${m.id}/${m.title}`">{{m.title}}</router-link>&nbsp;&nbsp; -->

<!-- 跳转路由并携带params参数，to的对象写法 -->
<router-link :to="{
	name:'xiangqing',
    params:{
        id:m.id,
        title:m.title
    }
}">
{{m.title}}
</router-link>

注意，params对象写法不能用path，要用name

# 9、router-link的replace属性

浏览器的历史记录为一个栈结构，每一次链接的更改都会进行入栈操作，指针也会默认的指向栈顶，当进行回退时，指针下移一格，前进操作时，指针上移一格，且不进行出栈操作。

而replace就是再进行入栈操作时，会进行出栈操作，像无痕浏览

  1. 作用：控制路由跳转时操作浏览器历史记录的模式

  2. 浏览器的历史记录有两种写入方式：分别为push和replace，push是追加历史记录，replace是替换当前记录。路由跳转时候默认为push

  3. 如何开启replace模式：<router-link replace .......>News</router-link>直接写replace即可

# 13、路由守卫

保护路由的安全，需要一定条件才能触发这个路由（登录和未登录）

跟路由器商量商量，在router暴露之前，做一个守卫

# 1、全局前置/后置路由守卫
//全局前置路由守卫————初始化的时候被调用、每次路由切换之前被调用
router.beforeEach((to,from,next)=>{
    // 所接受的有三个参数：去哪、来自哪、放行
    	// to和from是对象，分别是去和来的路由对象，有name、path等属性
  	// next() 一写就可以点击了，接下来就是判断什么时候放行，什么时候不放行
	console.log('前置路由守卫',to,from)
	if(to.meta.isAuth){ //判断是否需要鉴权
		if(localStorage.getItem('school')==='atguigu'){
            // 去localStorege中得到Item判断一下
			next()
		}else{
			alert('学校名不对，无权限查看！')
		}
	}else{
		next()
	}
})

//全局后置路由守卫————初始化的时候被调用、每次路由切换之后被调用
router.afterEach((to,from)=>{
    // 无next
	console.log('后置路由守卫',to,from)
	document.title = to.meta.title || '系统'
})

export default router
  • to.meta.isAuth即在一开始router配置中就给每一个routes配置一个特殊的属性，来标识是否可以跳转，是否需要校验权限
    • 但是如果直接添加一个属性，输出route时在route身上是看不到的，需要往meta这个属性里面装，因为meta里面有专门提供的容器供我们放一些我们想配置的信息
meta:{isAuth:true,title:'新闻'}

# 3、组件内路由守卫

在一个路由组件里直接写

<template>
	<h2>我是About的内容</h2>
</template>

<script>
	export default {
		name:'About',
		//通过路由规则，进入该组件时被调用
		beforeRouteEnter (to, from, next) {
			console.log('About--beforeRouteEnter',to,from)
			if(to.meta.isAuth){ //判断是否需要鉴权
				if(localStorage.getItem('school')==='atguigu'){
					next()
				}else{
					alert('学校名不对，无权限查看！')
				}
			}else{
				next()
			}
		},

		//通过路由规则，离开该组件时被调用
		beforeRouteLeave (to, from, next) {
			console.log('About--beforeRouteLeave',to,from)
			next()
            // 没放行就不能从这个路由组件出去！
		}
	}
</script>

# 14、路由器的两种基本模式

从 /#/到后面的路径都属于时hash值，其特点就是路径的改变的内容不会发给服务器，不涉及http请求

history工作模式：

总结：

  1. 对于一个url来说，什么是hash值？—— #及其后面的内容就是hash值。
  2. hash值不会包含在 HTTP 请求中，即：hash值不会带给服务器。
  3. hash模式：
    1. 地址中永远带着#号，不美观 。
    2. 若以后将地址通过第三方手机app分享，若app校验严格，则地址会被标记为不合法。
    3. 兼容性较好。
  4. history模式：
    1. 地址干净，美观 。
    2. 兼容性和hash模式相比略差。
    3. 应用部署上线时需要后端人员支持，解决刷新页面服务端404的问题。
