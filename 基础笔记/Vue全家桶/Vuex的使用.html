# 第二十四节 Vuex的使用

# 7.1搭建环境

  1. 先创建文件：src/store/index.js在js文件里这样写：为了顺利在index.js里创建并使用Vuex.Store({})，就不能在main.js里集中引入Vuex和use（Vuex），因为脚手架代码的执行顺序问题：会先将import语句解析完在执行其他语句

    //引入Vue核心库
import Vue from 'vue'
//引入Vuex
import Vuex from 'vuex'
//应用Vuex插件
Vue.use(Vuex)

//准备actions对象——响应组件中用户的动作
const actions = {}
//准备mutations对象——修改state中的数据
const mutations = {}
//准备state对象——保存具体的数据
const state = {}

//创建并暴露store
export default new Vuex.Store({
   actions,
   mutations,
   state
})

  2. 在main.js中创建vm时传入store配置项

    ......
//引入store
import store from './store'
......

//创建vm
new Vue({
   el:'#app',
   render: h => h(App),
   store
})

# 7.2基本使用

  1. 初始化数据、配置actions、配置mutations，操作文件store.js

    //引入Vue核心库
import Vue from 'vue'
//引入Vuex
import Vuex from 'vuex'
//引用Vuex
Vue.use(Vuex)

const actions = {
    //响应组件中加的动作
   jia(context,value){
      // console.log('actions中的jia被调用了',miniStore,value)
      context.commit('JIA',value)
   },
}

const mutations = {
    //执行加
   JIA(state,value){
      // console.log('mutations中的JIA被调用了',state,value)
      state.sum += value
   }
}

//初始化数据
const state = {
   sum:0
}

//创建并暴露store
export default new Vuex.Store({
   actions,
   mutations,
   state,
})

  2. 组件中读取vuex中的数据：$store.state.sum

  3. 组件中修改vuex中的数据：this.$store.dispatch('action中的方法名',数据) 或 this.$store.commit('mutations中的方法名',数据)

    备注：若没有网络请求或其他业务逻辑，组件中也可以越过actions，即不写dispatch，直接编写commit

Vuex由三个重要部分组成：

  • Actions：用于响应组件中的动作
  • Mutations：用于操作数据（state）vuex的开发者工具监视的也是Mutations
  • State：数据经state保管，其数据类型是一个对象（其实就是把data中的数据交给state这个属性保管）

组件们和Vuex构建联系

  • dispatch（Api）：调用时传递两个参数，第一个：动作类型，第二个：数据dispatch('jia',2)，相对应actions一定要保证有'jia'动作与之对应其值为一个函数，在这个函数里要写上commit('jia',2)，另外会通过一个“中间参数来调用commit这个函数”，这个中间参数有commit和dispatch、state等的属性供操作，如

    jiaOdd(context,value){
		console.log('actions中的jiaOdd被调用了')
		if(context.state.sum % 2){
			context.commit('JIA',value)
		}
	},

    ；通过commit将动作和数据进一步传递给Mutations，那么相对应Mutations里也有与之对应的函数jia function，且含有两个参数，第一个：state（因为这里真正的要对state的数据进行操作），第二个：数据2；之后可以写类似state.sum += value;其底层自动走的就是Mutate这条线到State；

  • 随后Vuex做的就是能重新解析这个组件去Render去渲染，于是页面上的sum也就变了

在这个过程中，Actions存在的特殊作用：当动作中要操作的数据涉及到Ajax等请求才能获取的时候，可以在actions里面去发送请求；当需要一定的业务逻辑时，比如什么条件下才加；其次如果用不到这个功能，vue允许能在组件里直接调用commit不走dispatch。

这三个对象都经store进行管理，因为组件和Vuex交流的api不是window提供的，而是store提供的，形式是$store.dispatch、$store.commit

使用Vuex的具体步骤

大致

1、安装Vuex

npm i vuex
//指令默认安装vuex4，而其只能在vue3中使用
npm i vuex@3

2、Vue.use(Vuex)

//在main.js里引用vuex
import vuex from 'vuex'
//引入之后使用
Vue.use(Vuex)
//引入，使用后的作用时，在创建vm的时候，能够配置store对象，使vm和vc身上有$store属性

3、配置store

目的是使store身上有dispatch和commit两个api还有Actions、Mutations和State三个属性，且能管理他们三个

在store文件夹下的js文件里写

//首先定义他们三个

//准备actions——用于响应组件中的动作
const actions = {
}
//准备mutations——用于操作数据（state）
const mutations = {
}
//准备state——用于存储数据
const state = {
}

//先引入Vuex
import Vuex from 'vuex' 
//这个store得通过一个东西new出来，就像vm,newVuex身上的一个构造函数Store(),并且创建的时候要传入配置对象
const store = new Vuex.Store({
    actions,
    mutations,
    state,
    //key和value重名可以简写
})

//最后暴露（导出）store
export default store

//之后在main.js里引用这个暴露出的store，再在vm配置这个store即可
import store from './store'

//创建vm
new Vue({
	el:'#app',
	render: h => h(App),
	store:store,
	beforeCreate() {
		Vue.prototype.$bus = this
	}
})

如果把vuex放在main.js里面use，因为vue脚手架解析代码的缘故，会优先解析import引入的文件，但是main里vuex还没有use，就会导致一个顺序错误而报错，更正:

//该文件用于创建Vuex中最为核心的store
import Vue from 'vue'
//先引入vue可以使用Vue.use()
//引入Vuex
import Vuex from 'vuex'
//应用Vuex插件
Vue.use(Vuex)

//准备actions——用于响应组件中的动作
const actions = {
}
//准备mutations——用于操作数据（state）
const mutations = {
}
//准备state——用于存储数据
const state = {
}

//创建并暴露store
export default new Vuex.Store({
	actions,
	mutations,
	state,
})

# 7.3 getters配置项

//该文件用于创建Vuex中最为核心的store
import Vue from 'vue'
//先引入vue可以使用Vue.use()
//引入Vuex
import Vuex from 'vuex'
//应用Vuex插件
Vue.use(Vuex)

//准备actions——用于响应组件中的动作
const actions = {
}
//准备mutations——用于操作数据（state）
const mutations = {
}
//准备state——用于存储数据
const state = {
}
//准备getters，用于将state中的数据进行加工
const getters = {
    bigSum(state){
        return state.sum*10
    }
}

//创建并暴露store
export default new Vuex.Store({
	actions,
	mutations,
	state,
    getters
})
  • getters是对state中的数据“再加工”，与vue中的计算属性相比，computed只能用于此组件内的使用，而getters能多组件使用，在配置项$store身上。组件中读取数据：$store.getters.bigSum

# 四个map方法总结

  • import {mapState,mapGetters,mapMutations,mapActions} from 'vuex'

  1. mapState方法：用于帮助我们映射state中的数据为计算属性

    computed: {
    //借助mapState生成计算属性：sum、school、subject（对象写法）
     ...mapState({sum:'sum',school:'school',subject:'subject'}),
         
    //借助mapState生成计算属性：sum、school、subject（数组写法）
    ...mapState(['sum','school','subject']),
},

  2. mapGetters方法：用于帮助我们映射getters中的数据为计算属性

    computed: {
    //借助mapGetters生成计算属性：bigSum（对象写法）
    ...mapGetters({bigSum:'bigSum'}),

    //借助mapGetters生成计算属性：bigSum（数组写法）
    ...mapGetters(['bigSum'])
},

  3. mapActions方法：用于帮助我们生成与actions对话的方法，即：包含$store.dispatch(xxx)的函数

    methods:{
    //靠mapActions生成：incrementOdd、incrementWait（对象形式）
    ...mapActions({incrementOdd:'jiaOdd',incrementWait:'jiaWait'})

    //靠mapActions生成：incrementOdd、incrementWait（数组形式）
    ...mapActions(['jiaOdd','jiaWait'])
}

  4. mapMutations方法：用于帮助我们生成与mutations对话的方法，即：包含$store.commit(xxx)的函数

    methods:{
    //靠mapActions生成：increment、decrement（对象形式）
    ...mapMutations({increment:'JIA',decrement:'JIAN'}),
    
    //靠mapMutations生成：JIA、JIAN（对象形式）
    ...mapMutations(['JIA','JIAN']),
}

备注：mapActions与mapMutations使用时，若需要传递参数需要：在模板中绑定事件时传递好参数，否则参数是事件对象。
