常见的Composition（组合） API

Keanu 4/22/2023 Vue3

# 第二节 常见的Composition（组合） API

关于什么是组合式API在后面会解释，这里先进行学习

# 1.拉开序幕的setup

  1. 理解：Vue3.0中一个新的配置项，值为一个函数。

  2. setup是所有Composition API（组合API）“ 表演的舞台 ”。

  3. 组件中所用到的：数据（data）、方法（computed、watch……）等等，均要配置在setup中。

    1. 直接写，直接定义变量和函数（方法）

  4. setup函数的两种返回值：

    1. 若返回一个对象，则对象中的属性、方法, 在模板中均可以直接使用。（重点关注！）

      • 在模板中直接插值语法使用
//返回一个对象（常用）
return {
    name,
    age,
    sayHello,
    test2,
    a
}

    2. 若返回一个渲染函数：则可以自定义渲染内容。（了解）

    3. 	export default {
		//此处只是测试一下setup，暂时不考虑响应式的问题。
		setup(){
			//数据
			let name = '张三'
			let age = 18
			let a = 200

			//方法
			function sayHello(){
				alert(`我叫${name}，我${age}岁了，你好啊！`)
			}
		}
	}

  5. 注意点：

    1. 尽量不要与Vue2.x配置混用
      • Vue2.x配置（data、methos、computed...）中可以访问到setup中的属性、方法。
      • 但在setup中不能访问到Vue2.x配置（data、methos、computed...）。
      • 如果有重名, setup优先。
    2. setup不能是一个async函数，因为返回值不再是return的对象, 而被promise包裹的对象, 模板不能替你.then拿到数据，即看不到return对象中的属性。（后期也可以返回一个Promise实例，但需要Suspense和异步组件的配合）

# 2.ref函数

  • 作用: 定义一个响应式的数据
  • 语法:
    • 先引用 import {ref} from 'vue'
    • 再定义 const xxx = ref(自己赋的值)
    • 原理是用ref方法，创建一个包含响应式数据的引用对象（reference对象，简称ref对象）。
    • JS中操作数据： xxx.value
    • 模板中读取数据: 不需要.value，模板能识别是RefImpl对象，自动.value，直接：<div></div>
  • 备注：
    • 接收的数据可以是：基本类型、也可以是对象类型。
    • 基本类型的数据：响应式依然是靠Object.defineProperty()的get与set完成的。
    • 对象类型的数据：内部 “ 求助 ” 了Vue3.0中的一个新函数—— reactive函数。

# 3.reactive函数

  • 作用: 定义一个对象类型的响应式数据（基本类型不要用它，要用ref函数）
  • 语法：const 代理对象= reactive(源对象)接收一个对象（或数组），返回一个代理对象（Proxy的实例对象，简称proxy对象）
  • reactive定义的响应式数据是“深层次的”。
  • 内部基于 ES6 的 Proxy 实现，通过代理对象操作源对象内部数据进行操作。

# 4.Vue3.0中的响应式原理

# vue2.x的响应式

  • 实现原理：

    • 对于对象类型的数据：是通过Object.defineProperty()对属性的读取、修改进行拦截（数据劫持：响应式的根基）。

    • 对于数组类型的数据：是通过重写更新数组的一系列方法来实现拦截。（对数组的变更方法进行了包裹）。

      Object.defineProperty(data, 'count', {
    get () {}, 
    set () {}
})

  • 存在问题：

    • 新增属性、删除属性, 界面不会更新。要手动添加set
      • this.$set(操作的对象，添加的属性名，属性值)
      • 或：Vue.set()
      • this.$delete(操作的对象，操作的属性)
    • 直接通过下标修改数组, 界面不会自动更新。

# Vue3.0的响应式

  • 实现原理:
    • 通过Proxy（代理）: 拦截对象中任意属性的变化, 包括：属性值的读写、属性的添加、属性的删除等。
    • 通过Reflect（反射）: 对源对象的属性进行操作。
    • MDN文档中描述的Proxy与Reflect：

      • Proxy：https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Proxy

      • Reflect：https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Reflect

        new Proxy(data, {
  // 拦截读取属性值
    get (target, prop) {
      return Reflect.get(target, prop)
    },
    // 拦截设置属性值或添加新属性
    set (target, prop, value) {
      return Reflect.set(target, prop, value)
    },
    // 拦截删除属性
    deleteProperty (target, prop) {
      return Reflect.deleteProperty(target, prop)
    }
})

proxy.name = 'tom'

# Vue3的响应式原理

# 1、模拟vue2实现响应式

// 源数据
let person = {
    name:"张三",
    age:18
}

// 模拟vue2实现响应式
let p = {}
Object.defineProperty(p,'name',{
    configurable:true,// 控制属性是否可以被删除，默认值为false
    get(){// 有人读取name调用
        return person.name
    },
    set(value){// 有人修改name式调用
        console.log('有人修改了name属性，我要去更新界面，这句话就是响应式的体现')
    	person.name = value
    }
})
// 可以选择循环遍历person对象，对其每一个属性进行监听

缺点：没有增加，删除对应的响应函数，且书写起来麻烦

# 2、vue3的响应式

需要认识window身上的一个人：window.Proxy内置的构造函数(代理)

const p = new Proxy(person,{})

// Proxy构造函数能让p去映射（代理）person的操作，参数第一个为对哪一个对象的代理,当第二个为一个空函数时，完成p对person对象的代理操作，即增删改查，但是还未实现响应式，还需要捕获到修改的数据，完成响应式

const p = new Proxy(person,{
    //有人读取p的某个属性时调用
    get(target,propName){
        // 靠所收到的两个参数完成后续操作，第一个为new proxy传入的对象，第二个为读取的是哪个属性（可能是name、可能是age……）
        console.log(`有人读取了p身上的${propName}属性`)
        
        return target.propName // 这样return是错误的❌
        1、这里的propName是形参 用.的方式就变成了实参 相当于你想在这里找一个键名为propName的属性
        2、用点的话相当于["propName"]
        3、因为propName是形参(变量)，形参直接点相当于找propName
        4、这里的target对象里面是没有propName属性的，所以用.的方式是找不到名为propName的属性的
        5、变量用[]、字符串用点或者[""]
        6、这么多人就一个人说对了，propName 是一个字符串，你见过 obj.'name' 这么操作对象的？
        
        return Reflect.get(target,propName)
    },
    //有人修改p的某个属性、或给p追加某个属性时调用
    set(target,propName,value){
        console.log(`有人修改了p身上的${propName}属性，我要去更新界面了！`)
        
        // target[propName] = value
        
        Reflect.set(target,propName,value)
    },
    //有人删除p的某个属性时调用
    deleteProperty(target,propName){
        console.log(`有人删除了p身上的${propName}属性，我要去更新界面了！`)
        
        // delete target[propName]
        // return delete target[propName]
        // 没有返回值将返回false
        
        return Reflect.deleteProperty(target,propName)
    }
})

window.ReflectReflect为window身上的一个属性，含多种方法可以实现对对象的操作

Reflect.get(obj,a)

……

现在，ECMA正在把Object身上的属性正在移植到Reflect身上，因为Object身上的一些方法比如.defineProperty会因为重名而报错

# 5.reactive对比ref

  • 从定义数据角度对比：
    • ref用来定义：基本类型数据。
    • reactive用来定义：对象（或数组）类型数据。
    • 备注：ref也可以用来定义对象（或数组）类型数据, 它内部会自动通过reactive转为代理对象。
  • 从原理角度对比：
    • ref通过Object.defineProperty()的get与set来实现响应式（数据劫持）。
    • reactive通过使用ES6新的构造函数：Proxy来实现响应式（或者叫：数据劫持）, 并通过Reflect（反射）来操作源对象内部的数据。
  • 从使用角度对比：
    • ref定义的数据：操作数据需要.value，读取数据时模板中直接读取不需要.value。
    • reactive定义的数据：操作数据与读取数据：均不需要.value。
  • 使用reactive，一般把所有数据由data对象包装，交给reactive保管

# 6.setup的两个注意点

  • setup执行的时机

    • 比beforeCreate还要早，在beforeCreate之前执行一次，且this是undefined（setup里面没有this）。

  • setup的参数

    • props：值为对象，包含：组件外部传递过来，且组件内部声明接收了的属性。

      • 但是这里setup接受到的props不是传统vue2props的作用，仍需要在setup外部写props配置项去接受传递的参数，只不过之后能在setup里面使用了**（还记得：vue2的props是只读的）**

    • context：上下文对象

      • attrs: 值为对象，包含：组件外部传递过来，但没有在props配置中声明的属性（如果在props配置项里声明了，那么attrs里就没有对象了）, 相当于 this.$attrs.外部传递的变量名。

      • slots: 收到的插槽内容, 相当于 this.$slots。

      • emit: 分发自定义事件的函数（用于触发自定义事件的）, 相当于 this.$emit。

        // 回忆自定义事件

// APP VUE中：
<template>
	<Demo @hello="showHelloMsg" msg="你好啊" school="尚硅谷">
		<template v-slot:qwe>
			<span>尚硅谷</span>
		</template>
		<template v-slot:asd>
			<span>尚硅谷</span>
		</template>
	</Demo>
</template>

<script>
	import Demo from './components/Demo'
	export default {
		name: 'App',
		components:{Demo},
		setup(){
			function showHelloMsg(value){
				alert(`你好啊，你触发了hello事件，我收到的参数是:${value}！`)
			}
			return {
				showHelloMsg
			}
		}
	}
</script>

// 子组件中：
<template>
	<h1>一个人的信息</h1>
	<h2>姓名：{{person.name}}</h2>
	<h2>年龄：{{person.age}}</h2>
	<button @click="test">测试触发一下Demo组件的Hello事件</button>
</template>

<script>
	import {reactive} from 'vue'
	export default {
		name: 'Demo',
		props:['msg','school'],
		emits:['hello'],
        // 传递自定义时间内时，需要用emits配置项告知一下传过来的事件
		setup(props,context){
			// console.log('---setup---',props)
			// console.log('---setup---',context)
			// console.log('---setup---',context.attrs) //相当与Vue2中的$attrs
			// console.log('---setup---',context.emit) //触发自定义事件的。
			console.log('---setup---',context.slots) //插槽
			//数据
			let person = reactive({
				name:'张三',
				age:18
			})

			//方法
			function test(){
				context.emit('hello',666)
			}

			//返回一个对象（常用）
			return {
				person,
				test
			}
		}
	}
</script>
